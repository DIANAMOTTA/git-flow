Git Flow

Branchig Model

Git Flow é um modelo de ramificação para organizar fluxo de trabalho de desenvolvimento de software.

O Git Flow define regras e comandos para criar e gerenciar branches com finalidades específicas, como:

  • master (ou main): versão de produção (código estável).
  • develop: versão de desenvolvimento (código em andamento).
  • feature/*: novas funcionalidades.
  • release/*: preparação para uma nova versão.
  • hotfix/*: correção urgente em produção.

    • Fluxo de Trabalho do Git Flow

  1. master —> produção
  2. develop —> base para features
  3. feature/ —> branch criada a partir de develop
  4. release/ —> criada a partir de develop, depois merge com master e develop
  5. hotfix/ —> criada a partir de master, depois merge com master e develop
